Configuring SAPT (GitHub Pages Source)

For all LadinOS versions (1.0+), add the following lines to your sources.list (default path: /etc/sapt/sources.list). This points SAPT to the GitHub Pages-hosted mirror:

sapt repo http://deb.saladin510-orangeyouhome-erath.online/ladinos stable main
sapt repo http://deb.saladin510-orangeyouhome-erath.online/ladinos-debug stable-debug main
sapt repo http://deb.saladin510-orangeyouhome-erath.online/ladinos-experimental unstable main

After saving the file, refresh SAPT’s package index to apply changes:

sudo sapt update

You can then install packages normally (e.g., sudo sapt install [package-name]).

GitHub Pages-Specific Notes

  • Static Asset Limitation: GitHub Pages serves static files (e.g., .deb, Packages.gz) — SAPT handles package index parsing and installation locally.
  • Update Delay: When new packages are added to the source, GitHub Pages may take 1–2 minutes to sync changes globally. If you don’t see updates immediately, wait a few minutes and re-run sudo sapt update.
  • Privacy & Security: All packages in /ladinos/ and /ladinos-debug/ are signed with LadinOS’s official GPG key. SAPT verifies these signatures by default to prevent tampered files.
  • Bandwidth: GitHub Pages provides unlimited bandwidth for public repositories — there’s no risk of hitting usage limits for personal or small-team use.

Troubleshooting

  • "404 Not Found" Errors: Ensure the repository path in sources.list matches (e.g., /ladinos/ — trailing slash required). Check that the GitHub Pages site is published (via your GitHub repo’s "Pages" settings).
  • Slow Downloads: GitHub’s CDN automatically routes you to the nearest node — if speeds are slow, try again later (CDN routing may take a moment to optimize).
  • SAPT Verification Errors: Re-import LadinOS’s GPG key with sudo sapt import-key official to fix signature verification issues.
